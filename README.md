# CentralAuth API

Multi-tenant authentication and authorization service with a React frontend.  
Built with .NET 8 + EF Core (MySQL) + React (Vite + TypeScript).

## Quick start (new development machine)

### 1. Clone

```bash
git clone <your-repo-url>
cd super-octo-engine-2
```

### 2. Configure the backend connection string

Edit `Central_auth_api/appsettings.Development.json`:

```json
"ConnectionStrings": {
  "CentralAuth": "Server=127.0.0.1;Port=3306;Database=centerl_auth;User=root;Password=<YOUR_MYSQL_PASSWORD>;AllowPublicKeyRetrieval=true;"
},
"Jwt": {
  "Key": "<at-least-32-char-random-string>"
}
```

> **Security tip:** Use `dotnet user-secrets` instead of storing passwords in config:
> ```bash
> dotnet user-secrets init --project Central_auth_api
> dotnet user-secrets set "ConnectionStrings:CentralAuth" "Server=127.0.0.1;Port=3306;Database=centerl_auth;User=root;Password=xxx;AllowPublicKeyRetrieval=true;" --project Central_auth_api
> dotnet user-secrets set "Jwt:Key" "your-secret-key" --project Central_auth_api
> ```

### 3. Create the database and apply migrations

```bash
cd Central_auth_api
dotnet ef database update
```

This creates the `centerl_auth` database with all tables in one shot from the single `InitialCreate` migration.

### 4. Start the API

```bash
dotnet run
```

The API starts at `http://localhost:5089` (HTTP) or `https://localhost:7170` (HTTPS).

**Health check:**

```bash
curl http://localhost:5089/health
# → {"status":"ok","service":"CentralAuth.Api","timestamp":"2026-06-06T..."}
```

**Swagger UI:**

Open `http://localhost:5089/swagger` in a browser.

### 5. Start the frontend

Open a second terminal:

```bash
cd Central_auth
npm install
npm run dev
```

The frontend starts at `http://localhost:5173` (Vite default).

> If the frontend gets CORS errors, add your Vite origin to `Central_auth_api/appsettings.Development.json`:
> ```json
> "Cors": {
>   "AllowedOrigins": ["http://localhost:5173"]
> }
> ```
> Then restart the API.

### 6. Verify end-to-end

- Open `http://localhost:5173` in a browser
- `POST /api/auth/login` with a registered user's credentials
- The login returns a JWT; subsequent requests include it via the **Authorize** button in Swagger UI

## Common tasks

### Drop and rebuild the database

```bash
dotnet ef database drop --force --project Central_auth_api
dotnet ef database update    --project Central_auth_api
```

### Generate a new migration (after model changes)

```bash
dotnet ef migrations add <MigrationName> --project Central_auth_api --output-dir Migrations
dotnet ef database update --project Central_auth_api
```

### Run the backend in Production mode

```bash
ASPNETCORE_ENVIRONMENT=Production dotnet run --project Central_auth_api
```

## Architecture notes

### Multi-tenant many-to-many

Users and tenants are linked through a join table (`auth_tenant_users` via the `TenantUser` entity). A single user login can belong to multiple tenants, each with:

- A unique `EmployeeId` per tenant (generated by `EmployeeIdGenerator`)
- Distinct roles and permissions per tenant (inherited via `UserRole` → `Role` → `RolePermission`)

### Unique constraints on the join table

| Index | Purpose |
|---|---|
| `IX_auth_tenant_users_AppUserId_TenantId` | A user can only be linked to the same tenant once |
| `IX_auth_tenant_users_TenantId_EmployeeId` | EmployeeId is unique within each tenant (nullable allowed) |

### Per-tenant EmployeeId generation

The `EmployeeIdGenerator` uses `SELECT ... FOR UPDATE` on the tenant row to serialize ID generation per tenant. Callers **must** wrap the call and the subsequent INSERT in the same `DbContext` transaction.

## API endpoints

| Method | Path | Description |
|---|---|---|
| `POST`   | `/api/auth/login` | Authenticate and receive JWT |
| `GET`    | `/api/users` | List users (paginated, filterable) |
| `POST`   | `/api/users` | Create user (one or more tenant links) |
| `PUT`    | `/api/users/{id}` | Update user profile + tenant links |
| `PATCH`  | `/api/users/{id}/lock` | Lock a user |
| `PATCH`  | `/api/users/{id}/unlock` | Unlock a user |
| `DELETE` | `/api/users/{id}` | Soft-delete a user |
| `GET`    | `/api/tenants` | List tenants |
| `POST`   | `/api/tenants` | Create tenant |
| `PUT`    | `/api/tenants/{id}` | Update tenant |
| `DELETE` | `/api/tenants/{id}` | Soft-delete tenant |
| `GET`    | `/api/roles` | List roles |
| `GET`    | `/api/permissions` | List permissions |
| `GET`    | `/api/modules` | List modules |
| `GET`    | `/api/departments` | List departments |
| `GET`    | `/api/designations` | List designations |
| `GET`    | `/api/dashboard/stats` | Dashboard statistics |
| `GET`    | `/api/health` | Health check |
| `GET`    | `/swagger` | Swagger UI (Development only) |
